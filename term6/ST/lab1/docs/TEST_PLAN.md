## 2. Тестируемый модуль

Класс: `ru.itmo.lab1.TanSeries`

Публичный API:
- `double TanSeries.tan(double x)`
- `double TanSeries.tan(double x, double eps, int maxTerms)`

Ключевые внутренние шаги (white-box точки для понимания):
- валидация входных данных (`validateInputs`)
- редукция аргумента к `[-pi/2, pi/2]` (`reduceToMinusHalfPiToHalfPi`)
- `sinSeries`, `cosSeries`
- `tan = sin / cos`
- защита от разрыва: если `|cos(x)| < eps`, выбрасывается `ArithmeticException`

## 3. Подход к тестированию

### 3.1 Техники
- Black-box тестирование:
  - разбиение на классы эквивалентности
  - анализ граничных значений
  - негативное тестирование (некорректные входные данные)
- Property-based проверки (математические свойства):
  - нечётность функции: `tan(-x) = -tan(x)`
  - периодичность: `tan(x + pi) = tan(x)`
- Сравнение с эталоном:
  - сравнение с `Math.tan(x)` на «безопасных» точках (далеко от `pi/2 + k*pi`)

### 3.2 Оракулы (ожидаемые результаты)
- Для безопасных точек: `TanSeries.tan(x)` ≈ `Math.tan(x)` в пределах допуска `delta`.
- Для точек, близких к разрывам: ожидается исключение `ArithmeticException`.
- Для некорректных входных данных: ожидается исключение `IllegalArgumentException`.

### 3.3 Допуски
По умолчанию:
- eps, используемый в функции: `1e-12`
- maxTerms: `200..500` в зависимости от теста
- `assert delta` (сравнение с `Math.tan`): `1e-10` (при необходимости можно ужесточить или ослабить)

## 4. Классы эквивалентности

### 4.1 Корректные классы входных данных (конечный x)
E1) Малый модуль: `x ∈ [-0.1, 0.1]`
- Ожидается очень быстрая сходимость и высокая точность.

E2) Средний модуль (безопасный): `x ∈ [-1.0, 1.0]`, исключая окрестности точек разрыва
- Типичный сценарий использования, устойчивое поведение.

E3) Вблизи границ интервала редукции: `x ≈ ±pi/2`, но не слишком близко
- Точность должна оставаться приемлемой; исключение выбрасываться не должно, если `|cos(x)| >= eps`.

E4) Большие значения |x|:
- Проверяется редукция аргумента: результаты должны соответствовать периодичности (`tan(x) = tan(x + pi*k)`).
- Примеры: `x = 1e6`, `x = -1e6`.

### 4.2 Некорректные классы входных данных
I1) x не является конечным числом: `NaN`, `+Infinity`, `-Infinity` -> `IllegalArgumentException`
I2) eps <= 0 или NaN -> `IllegalArgumentException`
I3) maxTerms <= 0 -> `IllegalArgumentException`

### 4.3 Класс разрыва
D1) `x` близок к `pi/2 + k*pi`:
- Если `|cos(x)| < eps`, ожидается `ArithmeticException`.

## 5. Граничные значения / критические точки

- x = 0
- x = ±pi/6, ±pi/4 (стандартные опорные углы)
- x близок к ±pi/2:
  - левая сторона: `pi/2 - δ`
  - правая сторона: `pi/2 + δ` (после редукции становится `-pi/2 + δ`)
- проверки периодичности:
  - x и x + pi
- большие x:
  - `x = 1e6`, `x = 1e6 + pi/3` (эквивалентно `pi/3` после редукции)

Выбранные значения δ:
- δ_safe = 1e-3 (должно быть безопасно, без исключения)
- δ_throw = 1e-12..1e-10 (может вызывать исключение в зависимости от eps); тесты строятся на условии `|cos(x)| < eps`.

## 6. Тест-кейсы (tg(x))

### 6.1 Корректность относительно эталона (Math.tan) — безопасные точки
TC-01: x = 0
- Ожидается: результат = 0 в пределах delta

TC-02: x = pi/6
- Ожидается: ≈ Math.tan(pi/6)

TC-03: x = -pi/4
- Ожидается: ≈ Math.tan(-pi/4) = -1

TC-04: x = 1.0
- Ожидается: ≈ Math.tan(1.0)

TC-05: x = -1.0
- Ожидается: ≈ Math.tan(-1.0)

### 6.2 Свойства
TC-06 (нечётность): взять x = 0.7
- Ожидается: tan(-x) ≈ -tan(x)

TC-07 (периодичность): взять x = -0.9
- Ожидается: tan(x + pi) ≈ tan(x)

### 6.3 Редукция аргумента (большие x)
TC-08: x = 1e6 + pi/6
- Ожидается: tan(x) ≈ tan(pi/6)

TC-09: x = -1e6 - pi/4
- Ожидается: tan(x) ≈ tan(-pi/4)

### 6.4 Поведение вблизи разрыва
TC-10 (безопасно у границы): x = pi/2 - 1e-3
- Ожидается: конечное значение, близкое к Math.tan(x) (может быть большим), без исключения

TC-11 (исключение в точке разрыва): x = pi/2
- Ожидается: ArithmeticException (cos ~ 0)

TC-12 (исключение в окрестности разрыва): x = pi/2 - tiny, где `|cos(x)| < eps`
- Ожидается: ArithmeticException

### 6.5 Негативные тесты (некорректные входные данные)
TC-13: x = NaN
- Ожидается: IllegalArgumentException

TC-14: x = +Infinity
- Ожидается: IllegalArgumentException

TC-15: eps = 0
- Ожидается: IllegalArgumentException

TC-16: maxTerms = 0
- Ожидается: IllegalArgumentException
